## 📘 1. exploratory_data_analysis.py — Полный EDA

### Что такое EDA?

**EDA (Exploratory Data Analysis)** — исследовательский анализ данных. Перед тем как строить модели, нужно **понять** данные: распределения, связи, выбросы, закономерности.

### Данные: интернет-магазин

```python
# 1000 заказов за 6 месяцев
sales_data = pd.DataFrame({
    'date': pd.date_range('2024-01-01', periods=1000, freq='D'),
    'product_category': np.random.choice(['Электроника', 'Одежда', 'Книги', 'Еда', 'Спорт'], 1000),
    'price': np.random.exponential(3000, 1000).clip(100, 20000),    # Цена товара
    'quantity': np.random.randint(1, 10, 1000),                       # Количество
    'customer_age': np.random.normal(35, 12, 1000).clip(18, 70),     # Возраст
    'customer_gender': np.random.choice(['М', 'Ж'], 1000),            # Пол
    'city': np.random.choice(['Минск', 'Гомель', ...], 1000),         # Город
    'discount_percent': np.random.choice([0, 5, 10, 15, 20], 1000),   # Скидка
    'is_return': np.random.choice([0, 1], 1000, p=[0.9, 0.1]),       # Возврат
    'rating': np.random.choice([1, 2, 3, 4, 5], 1000)                 # Оценка
})

# Выручка = цена × количество × (1 - скидка)
sales_data['revenue'] = sales_data['price'] * sales_data['quantity'] * (1 - sales_data['discount_percent'] / 100)
```

### ЧАСТЬ 7: Статистические тесты (5 тестов!)

#### Тест 1: T-тест (сравнение 2 групп)

```python
from scipy import stats

male_rev = sales_data[sales_data['gender'] == 'М']['revenue']
female_rev = sales_data[sales_data['gender'] == 'Ж']['revenue']

t_stat, p_value = stats.ttest_ind(male_rev, female_rev)
```

**Что проверяет:**
- **H₀ (нулевая гипотеза):** средние равны (нет разницы)
- **H₁ (альтернативная):** средние разные (есть разница)

**Результат:**
- `p < 0.05` → отвергаем H₀ → **разница значима**
- `p ≥ 0.05` → не отвергаем H₀ → **разница НЕ значима**

**Пример:**
```
Средняя выручка М: 4520.30 руб.
Средняя выручка Ж: 4480.15 руб.
p-value = 0.7234  →  разница НЕ значима (пол не влияет на выручку)
```

---

#### Тест 2: ANOVA (сравнение 3+ групп)

```python
city_groups = [group['revenue'].values for name, group in sales_data.groupby('city')]
f_stat, p_value = stats.f_oneway(*city_groups)
```

**Зачем:** T-тест сравнивает 2 группы. ANOVA — 3 и больше (у нас 6 городов).

**Что проверяет:**
- **H₀:** все средние равны (ни один город не отличается)
- **H₁:** хотя бы один отличается

**Пример:**
```
F-statistic: 2.45
p-value: 0.0312  →  разница значима! (α=0.05)
Вывод: хотя бы один город отличается
```

**Важно:** ANOVA говорит «кто-то отличается», но не говорит «кто именно». Для этого нужны post-hoc тесты (Тьюки).

---

#### Тест 3: Хи-квадрат (связь категориальных)

```python
contingency = pd.crosstab(sales_data['product_category'], sales_data['is_return'])
chi2, p_value, dof, expected = stats.chi2_contingency(contingency)
```

**Таблица сопряжённости:**

```
                is_return=0   is_return=1   Всего
Электроника        150           20         170
Одежда             130           30         160
Книги              180           10         190
Еда                200           5          205
Спорт              140           25         165
```

**Что проверяет:**
- **H₀:** категория и возвраты НЕ связаны
- **H₁:** связаны

**Пример:**
```
χ² = 18.45, p-value = 0.001  →  связь значима!
Вывод: одежда и электроника чаще возвращаются, чем еда
```

---

#### Тест 4: Шапиро-Уилка (нормальность)

```python
sample = sales_data['revenue'].sample(n=100, random_state=42)
stat, p_value = stats.shapiro(sample)
```

**Что проверяет:**
- **H₀:** данные распределены **нормально** (колокол Гаусса)
- **H₁:** НЕ нормально

**Пример:**
```
p-value = 0.0001  →  НЕ нормальное распределение
```

**Зачем:** Многие тесты (t-test, ANOVA) предполагают нормальность. Если данные НЕ нормальные, нужны непараметрические тесты (Манна-Уитни, Краскела-Уоллиса).

---

#### Тест 5: Корреляция Пирсона

```python
r, p_value = stats.pearsonr(sales_data['discount_percent'], sales_data['quantity'])
```

**Что проверяет:**
- **H₀:** корреляция = 0 (нет линейной связи)
- **H₁:** корреляция ≠ 0 (есть связь)

**Сила корреляции:**
- `|r| < 0.3` → слабая
- `0.3 ≤ |r| < 0.7` → средняя
- `|r| ≥ 0.7` → сильная

**Пример:**
```
r = 0.25, p-value = 0.0001  →  корреляция значима, но слабая
Вывод: скидка немного увеличивает количество, но слабо
```

### ЧАСТЬ 9: Доверительные интервалы

```python
def confidence_interval(data, confidence=0.95):
    n = len(data)
    mean = np.mean(data)
    sem = stats.sem(data)  # Стандартная ошибка среднего
    margin = sem * stats.t.ppf((1 + confidence) / 2, n - 1)
    return mean, margin, (mean - margin, mean + margin)

mean, margin, (ci_lower, ci_upper) = confidence_interval(sales_data['revenue'])
```

**Что это:** Мы не знаем **истинное** среднее генеральной совокупности. Но можем сказать: «с 95% уверенностью истинное среднее лежит в диапазоне [X, Y]».

**Пример:**
```
Средняя выручка: 4500.00 руб.
95% ДИ: (4350.00, 4650.00)
Погрешность: ±150.00
```

**Интерпретация:** Если бы мы взяли 100 выборок, в ~95 из них доверительный интервал содержал бы истинное среднее.

**Практическое применение:**
```
Город A: средняя выручка 5000 руб., 95% ДИ [4800, 5200]
Город B: средняя выручка 4700 руб., 95% ДИ [4400, 5000]

Интервалы ПЕРЕКРЫВАЮТСЯ → разница может быть случайной!
Нужен стат. тест (t-test), чтобы подтвердить значимость.
```

## 📘 2. ab_testing.py — A/B тестирование

### Условие

Компания запустила A/B тест новой функции в приложении. Нужно **проанализировать результаты** и дать **рекомендацию**.

### Данные

```python
data = pd.DataFrame({
    'user_id': range(1, 2001),
    'group': np.random.choice(['A', 'B'], 2000),        # A = контроль, B = новая функция
    'device': np.random.choice(['iOS', 'Android'], 2000),
    'day_of_week': np.random.randint(1, 8, 2000)
})

# Конверсия: B лучше на ~20%
conv_prob = np.where(data['group'] == 'B', 0.14, 0.11)  # A=11%, B=14%
data['converted'] = np.random.binomial(1, conv_prob)
```

---

### ШАГ 2: Основные метрики

```python
conversion = data.groupby('group').agg({
    'converted': ['sum', 'mean', 'count'],
    'session_duration': 'mean',
    'pages_viewed': 'mean'
})
```

**Результат:**
```
      converted_total  conversion_rate  total_users  avg_duration  avg_pages
A           115            0.1150          1000         10.2          5.1
B           142            0.1420          1000         12.5          6.8
```

**Lift (рост):**
```python
lift = (0.1420 - 0.1150) / 0.1150 = 0.235 = 23.5%
```
Версия B дала **23.5% рост** конверсии. Но это статистически значимо или случайность?

---

### ШАГ 3: Статистическая значимость (Z-тест)

```python
from statsmodels.stats.proportion import proportions_ztest

count = data.groupby('group')['converted'].sum()    # [142, 115]
nobs = data.groupby('group')['converted'].count()   # [1000, 1000]

z_stat, p_value = proportions_ztest(count.values, nobs.values)
```

**Результат:**
```
Z-statistic: 2.0156
P-value: 0.0439
Результат: ✓ ЗНАЧИМО (α = 0.05)
```

**Интерпретация:**
- `p = 0.0439 < 0.05` → разница **статистически значима**
- Версия B действительно лучше, а не повезло

---

### Доверительный интервал разницы (Бутстрап)

```python
def ci_diff_proportions(df, n_bootstraps=10000):
    diffs = []
    for _ in range(n_bootstraps):
        sample = df.sample(n=len(df), replace=True)  # Выборка с заменой
        p_a = sample[sample['group'] == 'A']['converted'].mean()
        p_b = sample[sample['group'] == 'B']['converted'].mean()
        diffs.append(p_b - p_a)

    alpha = 0.05
    lower = np.percentile(diffs, 100 * alpha / 2)     # 2.5-й перцентиль
    upper = np.percentile(diffs, 100 * (1 - alpha / 2))  # 97.5-й перцентиль
    return lower, upper

ci_lower, ci_upper = ci_diff_proportions(data)
```

**Как работает бутстрап:**
1. Берём исходную выборку (2000 пользователей)
2. Делаем **выборку с заменой** того же размера (некоторые пользователи попадут несколько раз, некоторые — ни разу)
3. Считаем разницу конверсий
4. Повторяем 10 000 раз
5. Берём 2.5-й и 97.5-й перцентили → 95% ДИ

**Результат:**
```
Разница конверсий: 0.0270
95% ДИ: (0.0021, 0.0520)
Вывод: ДИ не содержит 0 → значимо!
```

**Почему это важно:** Если ДИ содержит 0, значит разница может быть нулевой → результат не значим.

## 📊 Шпаргалка: какой тест когда использовать

| Что сравниваем | Тест | Пример |
|----------------|------|--------|
| 2 числовые группы | **T-тест** | М vs Ж, A vs B |
| 3+ числовые группы | **ANOVA** | 6 городов |
| 2 категориальные | **Хи-квадрат** | Категория ↔ Возвраты |
| 2 числовые (связь) | **Пирсон** | Скидка ↔ Количество |
| Нормальность | **Шапиро-Уилка** | Распределение выручки |
| Пропорции (A/B) | **Z-тест** | Конверсия A vs B |
