# 🎯 Кластеризация — Сегментация покупателей

---

## 🧠 Что такое кластеризация?

**Кластеризация** — это задача машинного обучения, где модель должна **найти естественные группы (кластеры) в данных** без заранее известных меток.

Простыми словами: модель смотрит на данные и отвечает на вопрос **«Какие группы здесь скрыты?»** или **«Как эти объекты группируются?»**.

### Ключевая идея

```
Входные данные (без меток!)  →  Модель  →  Кластеры (группы)

Примеры данных                 Результат кластеризации
─────────────────               ────────────────────────
1000 покупателей               Кластер 0: "Частые, мелкие покупки"
с разным поведением            Кластер 1: "Редкие, крупные покупки"
(частота, чек, время)          Кластер 2: "Активные исследователи"
                               Кластер 3: "Пассивные"

Точки на плоскости              Кластер A: точка в левом верхнем углу
(без подписей!)                 Кластер B: точка в правом нижнем углу
```

### Главное свойство кластеризации

> **У нас НЕТ правильных ответов (меток). Модель САМА находит структуру в данных.**

Это называется **обучение без учителя (unsupervised learning)**. В отличие от классификации и регрессии, где модель учится на примерах «вот это — спам, вот это — не спам», в кластеризации модель говорит: «я вижу, что данные разбиваются на 3 группы, вот они».

### Чем кластеризация отличается от классификации?

```
КЛАССИФИКАЦИЯ (обучение С учителем):
  Дано: 1000 писем, каждое помечено как "спам" или "не спам"
  Модель учит: "спам содержит слова 'выигрыш', 'бесплатно'"
  Новое письмо → модель говорит: "это спам"

КЛАСТЕРИЗАЦИЯ (обучение БЕЗ учителя):
  Дано: 1000 писем БЕЗ меток
  Модель ищет: "письма с короткими фразами группируются отдельно,
                письма с ссылками — отдельно"
  Результат: 2 кластера, но модель НЕ знает, какой из них "спам"
```

### Типы кластеризации

| Метод | Как работает | Когда использовать |
|-------|-------------|-------------------|
| **K-Means** | Находит k центров кластеров, назначает точки ближайшему | Знаю число k, кластеры сферические |
| **DBSCAN** | Ищет плотные области, помечает выбросы как "шум" | Не знаю k, кластеры произвольной формы, есть выбросы |
| **Иерархическая** | Постепенно объединяет ближайшие точки в группы | Нужна иерархия (дерево кластеров), мало данных |
| **Agglomerative** | Снизу вверх: каждая точка → кластер, потом объединяются | Альтернатива K-Means, когда форма кластеров неизвестна |

### Где применяется?

| Область | Задача | Что находит |
|---------|--------|-------------|
| 🛒 Магазины | Сегментация покупателей | Типы клиентов для маркетинга |
| 🎵 Музыка | Группировка песен по жанрам | Рок, поп, акустика (без меток!) |
| 🏙️ Геоданные | Группировка городов по активности | Мегаполисы, средние, малые |
| 🧬 Биология | Классификация видов по ДНК | Эволюционные группы |
| 🖼️ Изображения | Сжатие цветов | k основных цветов в картинке |
| 📝 Тексты | Тематическая группировка документов | Группы по темам |
| 🔐 Безопасность | Обнаружение аномалий | Выбросы = подозрительная активность |

### Классификация vs Регрессия vs Кластеризация

```
┌─────────────────┬──────────────────┬──────────────────┬──────────────────┐
│                 │  КЛАССИФИКАЦИЯ   │   РЕГРЕССИЯ      │  КЛАСТЕРИЗАЦИЯ   │
├─────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Вопрос          │ «Что это?»       │ «Сколько?»       │ «Какие группы?»  │
│ Ответ           │ Категория        │ Число            │ Группа (кластер) │
│ Учителя?        │ ✅ Да (есть y)   │ ✅ Да (есть y)   │ ❌ Нет (нет y)   │
│ Пример          │ Спам / Не спам   │ Цена = 15000     │ Группа покупателей│
│ Метод           │ Логистическая    │ Линейная         │ K-Means          │
│                 │ регрессия        │ регрессия        │ DBSCAN           │
└─────────────────┴──────────────────┴──────────────────┴──────────────────┘
```

### Как работает K-Means (самый популярный метод)?

```
Шаг 1: Случайно размещаем k центроидов (центров кластеров)
       ×                    ×

Шаг 2: Назначаем каждую точку ближайшему центроиду
       •••• ×       × ••••

Шаг 3: Перемещаем центроиды в средний центр своих точек
              ••••
         ••••  ×       ×  ••••
              ••••

Шаг 4: Повторяем шаги 2-3, пока центроиды не стабилизируются
       (перестанут двигаться)

Итог: k кластеров, каждый со своим центром
```

### Как найти число кластеров (k)?

В реальности мы **не знаем**, сколько групп в данных. Два основных метода:

**1. Метод локтя (Elbow Method):**
```
k=2: inertia = 1200  ─┐
k=3: inertia = 800    ─┤ резкий спад
k=4: inertia = 450    ─┘ ← "локоть" — здесь оптимальное k
k=5: inertia = 380    ─┐
k=6: inertia = 320    ─┤ медленный спад
k=7: inertia = 280    ─┘
```

**2. Silhouette Score (силуэтный коэффициент):**
```
Для каждой точки:
  a = среднее расстояние до точек СВОЕГО кластера (ближе = лучше)
  b = среднее расстояние до точек БЛИЖАЙШЕГО чужого кластера (дальше = лучше)
  silhouette = (b - a) / max(a, b)

Результат от -1 до +1:
  +1 → точка отлично в своём кластере
   0 → точка на границе
  -1 → точка, скорее всего, в чужом кластере

Чем выше средний silhouette → тем лучше кластеризация!
```

```python
# Группа 1: "Частые, но дешёвые покупки" (50 чел.)
group1 = pd.DataFrame({

group2 = pd.DataFrame({
    'frequency': np.random.normal(2, 1, 40),      # 2 покупки/мес — редко
    'avg_check': np.random.normal(5000, 1000, 40), # 5000 руб. — крупно
    ...
})

# Группа 3: "Активные исследователи" (60 чел.)
group3 = pd.DataFrame({
    'frequency': np.random.normal(8, 2, 60),
    'avg_check': np.random.normal(2000, 500, 60),
    'time_on_site': np.random.normal(300, 50, 60), # Много времени на сайте
    'pages_viewed': np.random.normal(80, 15, 60)   # Много страниц

# Группа 4: "Пассивные" (30 чел.)
group4 = pd.DataFrame({

data = pd.concat([group1, group2, group3, group4], ignore_index=True)
```

**Зачем:** В реальности данные НЕ размечены. Но для обучения мы генерируем с известными метками, чтобы проверить, правильно ли модель нашла группы.

### ШАГ 2: Масштабирование (КРИТИЧЕСКИ ВАЖНО!)

```python
scaler = StandardScaler()
data_scaled = scaler.fit_transform(data)
```

**Почему ОБЯЗАТЕЛЬНО:** Кластеризация использует **расстояния** между точками.

**Без масштабирования:**
- `frequency` = 2-15 (диапазон ~13)
- `avg_check` = 500-5000 (диапазон ~4500)

Признак `avg_check` будет **доминировать** — расстояние определяется им, остальные игнорируются.

**После масштабирования:** Все признаки имеют `mean=0, std=1` → равный вклад.

### ШАГ 3: Как найти число кластеров?

В реальности мы **не знаем**, сколько кластеров. Два метода:

#### Метод 1: Elbow Method (Метод локтя)

```python
inertias = []
for k in range(2, 11):
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(data_scaled)
    inertias.append(kmeans.inertia_)
```

**Inertia (WCSS)** = сумма квадратов расстояний от точек до центров кластеров. Чем больше k, тем меньше inertia.

**Как выбрать k:** Ищем «локоть» — точку, где убывание резко замедляется.

```
k=2:  inertia = 1200  ─┐
k=3:  inertia = 800    ─┤ резкий спад
k=4:  inertia = 450    ─┘
k=5:  inertia = 380    ─┐
k=6:  inertia = 320    ─┤ медленный спад → «локоть» ở k=4
k=7:  inertia = 280    ─┘
```

#### Метод 2: Silhouette Score (Силуэтный коэффициент)

```python
for k in range(2, 11):
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = kmeans.fit_predict(data_scaled)
    score = silhouette_score(data_scaled, labels)
    silhouette_scores.append(score)
```

**Как работает:** Для каждой точки вычисляет:
- `a` = среднее расстояние до точек **своего** кластера (ближе = лучше)
- `b` = среднее расстояние до точек **ближайшего чужого** кластера (дальше = лучше)
- `silhouette = (b - a) / max(a, b)`

**Значения:**
- `+1` → точка отлично в своём кластере
- `0` → точка на границе
- `-1` → точка, скорее всего, в чужом кластере

**Чем выше средний silhouette → тем лучше k.**

### ШАГ 4: K-Means

```python
kmeans = KMeans(n_clusters=4, random_state=42, n_init=10, max_iter=300)
kmeans_labels = kmeans.predict(data_scaled)
```

**Как работает алгоритм:**

```
1. Случайно размещаем 4 центроида (центра кластеров)
2. Назначаем каждую точку ближайшему центроиду
3. Перемещаем центроиды в средний центр своих точек
4. Повторяем шаги 2-3, пока центроиды не стабилизируются

Итог: 4 кластера, каждый с центром
```

**Результат:**

```
Кластер 0 (50 чел.): "Частые покупки (мелкие)"
  Частота: 15.2, Чек: 498 руб., Возвраты: 5.1%

Кластер 1 (40 чел.): "Редкие покупки (крупные)"

Кластер 2 (60 чел.): "Активные исследователи"
  Частота: 8.0, Чек: 2010 руб., Время: 298 мин

Кластер 3 (30 чел.): "Пассивные клиенты"
  Частота: 1.0, Чек: 805 руб., Время: 31 мин
```

**Метрики:**
- `silhouette_score` → 0.55 (хорошее разделение)
- `calinski_harabasz_score` → 350 (чем больше, тем лучше)

### ШАГ 5: DBSCAN

```python
dbscan = DBSCAN(eps=0.5, min_samples=5)
dbscan_labels = dbscan.fit_predict(data_scaled)
```

**Как работает:**

```
1. Для каждой точки смотрим окрестность радиуса eps
2. Если в окрестности ≥ min_samples точек → это «ядро» кластера
3. Все точки в окрестности ядра → тот же кластер
4. Точки, не попавшие ни в один кластер → ВЫБРОСЫ (label = -1)
```

**Параметры:**
- `eps` = радиус окрестности. Малый → много маленьких кластеров. Большой → один большой.
- `min_samples` = минимум точек для кластера. Больше → строже.

**Как подобрать eps:** K-distance graph — строим расстояния до k-го соседа, ищем «перегиб».

```python
from sklearn.neighbors import NearestNeighbors

k = 5
nn = NearestNeighbors(n_neighbors=k)
nn.fit(data_scaled)
distances, _ = nn.kneighbors(data_scaled)
distances = np.sort(distances[:, -1])

# График: ось X = точки, ось Y = расстояние до 5-го соседа
# Ищем резкий перегиб → значение в этой точке = хороший eps
```

**Отличие от K-Means:**
| K-Means | DBSCAN |
|---------|--------|
| Нужно знать k | k определяется автоматически |
| Кластеры — сферы | Кластеры любой формы |
| Все точки в кластере | Есть выбросы (label = -1) |
| Чувствителен к начальным центроидам | Детерминированный |

### ШАГ 6: Иерархическая кластеризация

```python
agg = AgglomerativeClustering(n_clusters=4)
agg_labels = agg.fit_predict(data_scaled)
```

**Как работает (агломеративный подход):**

```
1. Каждая точка — отдельный кластер (180 кластеров)
2. Находим 2 ближайших кластера → объединяем
3. Повторяем, пока не останется 4 кластера
```

**Плюсы:** Не нужно задавать центроиды, можно строить дендрограмму (дерево кластеров).
**Минусы:** Медленная на больших данных (O(n²)).

### ШАГ 9: Профилирование — даём имена кластерам

```python
def name_cluster(row):
    # Нормализуем относительно средних по всем данным
    freq_norm = (row['frequency'] - data['frequency'].mean()) / data['frequency'].std()
    check_norm = (row['avg_check'] - data['avg_check'].mean()) / data['avg_check'].std()
    time_norm = (row['time_on_site'] - data['time_on_site'].mean()) / data['time_on_site'].std()

    if freq_norm > 0.5 and check_norm < -0.5:
        return "Частые покупки (мелкие)"
    elif freq_norm < -0.5 and check_norm > 0.5:
        return "Редкие покупки (крупные)"
    elif time_norm > 0.5:
        return "Активные исследователи"
    else:
        return "Пассивные клиенты"
```

**Бизнес-применение:**
- «Частые мелкие» → предложить подписку, скидку за объём
- «Редкие крупные» → программа лояльности, персональные предложения
- «Исследователи» → уже вовлечены, можно продавать больше
- «Пассивные» → re-marketing, email-рассылка, скидки

## 📊 Метрики кластеризации

| Метрика | Диапазон | Что означает | Лучше |
|---------|----------|-------------|-------|
| **Silhouette** | [-1, 1] | Насколько точки «свои» в кластере | Больше |
| **Calinski-Harabasz** | [0, +∞) | Отношение меж-/внутрикластерной дисперсии | Больше |
| **Inertia** | [0, +∞) | Сумма квадратов расстояний до центров | Меньше |

**Важно:** Inertia всегда уменьшается с ростом k → нельзя использовать для выбора k! Silhouette — можно.

---

## 🛠 Когда какой метод использовать?

| Ситуация | Метод | Почему |
|----------|-------|--------|
| Знаю k, сферические кластеры | **K-Means** | Быстрый, простой |
| Не знаю k, возможны выбросы | **DBSCAN** | Сам находит k, маркирует шум |
| Мало данных, нужна иерархия | **Иерархическая** | Дендрограмма, наглядность |
| Очень много данных | **MiniBatchKMeans** | Быстрее обычного K-Means |
| Кластеры разной размера/плотности | **DBSCAN** | K-Means предполагает одинаковые |
